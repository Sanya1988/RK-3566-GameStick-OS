# GameStick OS User Manual

## 1. Назначение

Это пользовательское руководство по работе с GameStick OS.

В этом документе описано:
- как проходит первый запуск стика
- как устроена карта памяти
- куда класть ROM-файлы, BIOS, музыку, темы, читы и картинки заставки
- как работает ES-DE
- как пользоваться RetroArch во время игры
- где искать сохранения, save states и скриншоты
- как правильно добавлять медиа для игр
- что делать, если что-то не видно или не запускается

Документ намеренно написан без технических подробностей сборки.

## 2. Как устроен стик

После записи образа на карту и первого запуска система использует два раздела:

- `rootfs`
  Системный раздел. Пользователь обычно его не меняет.
- `USERDATA`
  Пользовательский раздел. Именно с ним вы будете работать на компьютере.

Все ROM-файлы, BIOS, музыка, темы, медиа, читы, сохранения и скриншоты находятся в `USERDATA`.

Если открыть карту на компьютере, работать нужно именно с разделом `USERDATA`.

## 3. Первый запуск

На первом запуске GameStick OS готовит раздел `USERDATA`.

Текущее поведение системы:

1. При первом включении запускается `Этап 1`.
2. После завершения первого этапа стик выключается.
3. Нужно включить стик повторно.
4. При втором включении запускается `Этап 2`.
5. После завершения второго этапа открывается ES-DE.

Это нормальное поведение для данной версии системы.

Во время обоих этапов:
- не отключайте питание
- не вынимайте карту памяти

## 4. Что уже есть в системе после первого запуска

После первичной подготовки система сама создаёт структуру папок в `USERDATA`.

Также в системе уже будут:
- папки для ROM-файлов
- папка для BIOS
- папка для фоновой музыки
- папка для заставки ES-DE
- папки ES-DE
- папки RetroArch
- несколько демо-игр

## 7. Как добавить ROM-файлы

Самый простой способ:

1. Полностью выключите стик.
2. Извлеките карту памяти.
3. Откройте раздел `USERDATA` на компьютере.
4. Перейдите в `/roms/<нужная_система>`.
5. Скопируйте туда ROM-файлы.
6. Безопасно извлеките карту.
7. Вставьте карту обратно в стик.
8. Запустите стик.

Если игры не появились сразу, проще всего перезапустить ES-DE через меню выхода.

Важно:
- папка `REBOOT SYSTEM` в текущей сборке перезапускает именно ES-DE, а не весь стик
- для полного выключения используйте `POWER OFF SYSTEM`

## 8. Демо-игры

В системе уже добавлены бесплатные демо-игры, чтобы после первого запуска стик не был пустым.

Они лежат в соответствующих системных папках и служат примером того, как должна выглядеть рабочая структура.

## 9. BIOS-файлы

BIOS нужно класть в:

- `/bios`

Некоторые BIOS лежат прямо в корне `/bios`, а некоторые используют подпапки.

Примеры популярных BIOS-файлов:

- `/bios/gba_bios.bin`
- `/bios/disksys.rom`
- `/bios/neogeo.zip`
- `/bios/syscard3.pce`
- `/bios/scph5500.bin`
- `/bios/scph5501.bin`
- `/bios/scph5502.bin`
- `/bios/bios_CD_E.bin`
- `/bios/bios_CD_U.bin`
- `/bios/bios_CD_J.bin`

Также есть системы, для которых нужны целые наборы файлов и подпапок, например:

- Amiga
- MSX
- CD-системы

Рекомендации:

- не переименовывайте BIOS без необходимости
- сохраняйте точные имена файлов
- если BIOS поставляется вместе с подпапками, переносите структуру целиком

Важно:
- BIOS не входят в публичный release по юридическим причинам
- если игра не запускается, одна из первых причин почти всегда связана с отсутствующим или неправильно названным BIOS

## 10. ES-DE: что это такое

ES-DE — это главное графическое меню системы.

Именно в нём вы:
- выбираете систему
- выбираете игру
- видите описание, обложки, видео и прочее медиа
- настраиваете тему
- включаете музыку
- задаёте таймер заставки
- настраиваете управление

## 11. Первичная настройка геймпада

Если пользовательская раскладка ещё не создана, ES-DE при запуске открывает мастер настройки управления.

Это нормальное поведение.

Текущее поведение системы:
- раскладка не навязывается заранее
- пользователь задаёт кнопки сам
- визуальный тип кнопок в интерфейсе по умолчанию настроен под `PlayStation`

Это значит:
- подсказки в интерфейсе будут показываться в стиле `PS`
- физически вы можете использовать и другой геймпад

## 12. Как заново вызвать настройку геймпада

Если вы хотите перенастроить геймпад с нуля:

1. Откройте раздел `USERDATA`.
2. Перейдите в `/esde/settings`.
3. Удалите файл:
   `/esde/settings/es_input.xml`
4. Запустите ES-DE снова.

После этого мастер настройки откроется заново.

## 13. Главное меню ES-DE

Основные разделы, с которыми пользователь обычно работает:

- `UI SETTINGS`
- `SOUND SETTINGS`
- `INPUT DEVICE SETTINGS`
- `SCREENSAVER SETTINGS`
- `OTHER SETTINGS`
- `QUIT`

Полезные пункты:

- `UI SETTINGS -> THEME`
  Выбор темы ES-DE
- `SOUND SETTINGS -> ENABLE BACKGROUND MUSIC`
  Включение и выключение фоновой музыки
- `SOUND SETTINGS -> BACKGROUND MUSIC VOLUME`
  Громкость фоновой музыки
- `INPUT DEVICE SETTINGS -> CONTROLLER TYPE`
  Визуальный тип кнопок
- `SCREENSAVER SETTINGS`
  Настройка заставки

## 14. Выход, перезапуск и выключение

В меню выхода доступны важные действия.

Текущее поведение системы:

- `REBOOT SYSTEM`
  Перезапускает ES-DE
- `POWER OFF SYSTEM`
  Корректно выключает стик

Если вам нужно, чтобы ES-DE перечитал новые игры, темы, медиа или настройки, обычно достаточно `REBOOT SYSTEM`.

## 15. Фоновая музыка в ES-DE

Фоновая музыка читается из:

- `/music`

Поддерживаемые форматы:

- `.aac`
- `.flac`
- `.m4a`
- `.mp3`
- `.ogg`
- `.opus`
- `.wav`

Что можно сделать:

1. Скопировать свои музыкальные файлы в `/music`.
2. В ES-DE открыть `SOUND SETTINGS`.
3. Включить `ENABLE BACKGROUND MUSIC`.
4. Настроить `BACKGROUND MUSIC VOLUME`.

Переключение треков в ES-DE:

- `L3` — предыдущий трек
- `R3` — следующий трек

На клавиатуре:

- `F2` — предыдущий трек
- `F3` — следующий трек

Важно:
- это относится к ES-DE
- внутри игры музыка ES-DE не управляет RetroArch и не заменяет звук игры

## 16. Экранная заставка ES-DE

Картинки заставки хранятся в:

- `/screensaver`

Поддерживаемые форматы:

- `.jpg`
- `.png`

Рекомендуемое разрешение:

- `1920x1080`

Как это работает:

- ES-DE показывает все изображения, которые лежат в `/screensaver`
- смена картинок происходит автоматически
- используется слайд-шоу
- переход между картинками делается плавно

Как настроить:

1. Откройте `SCREENSAVER SETTINGS`.
2. Включите тип заставки `SLIDESHOW`, если он был изменён.
3. Настройте `START SCREENSAVER AFTER (MINUTES)`.
4. При необходимости проверьте, что включён режим `USE CUSTOM IMAGES`.

Примечание:
- в публичном release папка может быть пустой или содержать только `README.txt`
- вы можете положить в неё свои картинки вручную

## 17. Темы ES-DE

Темы нужно класть сюда:

- `/esde/themes`

Правильная структура:

- каждая тема в отдельной подпапке
- внутри папки темы должен быть файл `theme.xml`

Пример:

- `/esde/themes/MyTheme/theme.xml`

После копирования темы:

1. Откройте `UI SETTINGS`.
2. Перейдите в `THEME`.
3. Выберите новую тему.

Официальный список тем ES-DE:

- https://gitlab.com/es-de/themes/themes-list

Важно:
- онлайн-обновление тем в этой системе не является основным способом установки
- надёжнее копировать тему вручную в `/esde/themes`

## 18. Gamelist в ES-DE

В этой сборке ES-DE читает `gamelist.xml` прямо из папки системы.

Правильный путь:

- `/roms/<system>/gamelist.xml`

Примеры:

- `/roms/megadrive/gamelist.xml`
- `/roms/nes/gamelist.xml`
- `/roms/snes/gamelist.xml`

В `gamelist.xml` имеет смысл хранить:

- название игры
- описание
- жанр
- дату
- рейтинг
- издателя
- прочие текстовые метаданные

Важно:
- в этой сборке ES-DE не полагается на медиа-теги внутри `gamelist.xml`
- картинки и видео нужно класть в специальную папку `downloaded_media`

## 19. Как правильно добавлять медиа для игр

Медиа ES-DE хранится здесь:

- `/esde/downloaded_media`

Структура такая:

- `/esde/downloaded_media/<system>/covers`
- `/esde/downloaded_media/<system>/screenshots`
- `/esde/downloaded_media/<system>/fanart`
- `/esde/downloaded_media/<system>/marquees`
- `/esde/downloaded_media/<system>/titlescreens`
- `/esde/downloaded_media/<system>/miximages`
- `/esde/downloaded_media/<system>/3dboxes`
- `/esde/downloaded_media/<system>/backcovers`
- `/esde/downloaded_media/<system>/physicalmedia`
- `/esde/downloaded_media/<system>/custom`
- `/esde/downloaded_media/<system>/videos`

Поддерживаемые форматы изображений:

- `.png`
- `.jpg`
- `.webp`

Поддерживаемые форматы видео:

- `.mp4`
- `.mkv`
- `.avi`
- `.wmv`
- `.mov`
- `.webm`

Главное правило:

- имя медиа должно совпадать с именем ROM-файла без расширения

Пример:

Если ROM лежит здесь:

- `/roms/megadrive/Sonic The Hedgehog.bin`

то обложка должна лежать, например, здесь:

- `/esde/downloaded_media/megadrive/covers/Sonic The Hedgehog.png`

а скриншот здесь:

- `/esde/downloaded_media/megadrive/screenshots/Sonic The Hedgehog.png`

а видео здесь:

- `/esde/downloaded_media/megadrive/videos/Sonic The Hedgehog.mp4`

## 20. Если ROM лежит в подпапке

Если ROM лежит не в корне системной папки, а глубже, структуру нужно повторять и в медиа.

Пример:

ROM:

- `/roms/megadrive/hacks/Sonic Hack.bin`

Тогда медиа:

- `/esde/downloaded_media/megadrive/covers/hacks/Sonic Hack.png`
- `/esde/downloaded_media/megadrive/screenshots/hacks/Sonic Hack.png`
- `/esde/downloaded_media/megadrive/videos/hacks/Sonic Hack.mp4`

## 21. Что делать после добавления новых игр и медиа

После копирования новых ROM-файлов, `gamelist.xml` или медиа:

1. Безопасно извлеките карту памяти из компьютера.
2. Вставьте карту в стик.
3. Запустите ES-DE.
4. Если изменения не появились, используйте `REBOOT SYSTEM` в меню выхода.

## 22. RetroArch: что это такое

Большинство игр в системе запускается через RetroArch.

RetroArch отвечает за:

- запуск ядер
- эмуляцию
- меню во время игры
- сохранения
- save states
- читы
- скриншоты
- ремапы
- шейдеры

## 23. Где RetroArch хранит свои данные

Основные пользовательские папки RetroArch:

| Папка | Назначение |
|---|---|
| `/retroarch/cores` | Ядра эмуляторов |
| `/retroarch/info` | Информация о ядрах |
| `/retroarch/cheats` | Читы |
| `/retroarch/remaps` | Переназначение кнопок |
| `/retroarch/saves` | Обычные сохранения |
| `/retroarch/states` | Save states |
| `/retroarch/screenshots` | Скриншоты |
| `/retroarch/thumbnails` | Миниатюры |
| `/retroarch/overlays` | Overlay-файлы |
| `/retroarch/shaders` | Шейдеры |

Важно:
- не редактируйте папку `/retroarch/cores` без необходимости
- не удаляйте оттуда файлы
- замена ядер случайными версиями из интернета может сломать запуск игр

## 24. Как открыть меню RetroArch во время игры

Здесь есть важный нюанс.

В этой сборке способ вызова меню зависит от профиля геймпада.

Практически это выглядит так:

- на PlayStation-совместимых геймпадах меню обычно открывается центральной кнопкой `PS/Guide`
- на других геймпадах поведение может зависеть от конкретного профиля

Если меню не открывается:

1. Попробуйте центральную системную кнопку геймпада.
2. Если это не сработало, зайдите в RetroArch и переназначьте вызов меню вручную:
   `Settings -> Input -> Hotkeys -> Menu Toggle`

Самое безопасное правило:

- не рассчитывайте на случайные комбинации
- лучше один раз проверить, чем именно открывается меню на вашем конкретном геймпаде

## 25. Как выйти из игры правильно

Самый надёжный способ:

1. Откройте меню RetroArch.
2. Перейдите в `Quick Menu`.
3. Выберите `Close Content` или `Quit RetroArch`, в зависимости от нужного результата.

Если вы не уверены в горячих клавишах вашего геймпада, выход через меню — самый безопасный вариант.

## 26. Обычные сохранения и save states

В RetroArch есть два разных типа сохранений.

### Обычные внутриигровые сохранения

Это сохранения, которые делает сама игра.

Они хранятся здесь:

- `/retroarch/saves`

### Save states

Это моментальные снимки состояния эмулятора.

Они хранятся здесь:

- `/retroarch/states`

Разница:

- обычное сохранение зависит от самой игры
- save state работает на уровне эмулятора

Если хотите максимальную совместимость, лучше пользоваться обычными игровыми сохранениями, когда игра их поддерживает.

## 27. Скриншоты

Скриншоты RetroArch сохраняются сюда:

- `/retroarch/screenshots`

Если вы сделали снимок во время игры, ищите его именно там.

## 28. Читы RetroArch

Читы нужно класть сюда:

- `/retroarch/cheats`

Формат файлов:

- `.cht`

Разрешается:

- хранить `.cht` в корне `/retroarch/cheats`
- создавать подпапки по системам
- создавать подпапки по играм

Официальный архив читов RetroArch:

- https://buildbot.libretro.com/assets/frontend/cheats.zip

Важно:
- онлайн-обновление RetroArch в этой сборке отключено
- поэтому читы нужно скачивать и копировать вручную

## 29. Как включить чит в игре

Один из рабочих вариантов:

1. Запустите игру.
2. Откройте меню RetroArch.
3. Перейдите в `Quick Menu`.
4. Откройте раздел `Cheats`.
5. Выберите `Load Cheat File (Replace)` или `Load Cheat File (Append)`.
6. Найдите нужный `.cht` файл.
7. После загрузки включите нужные чит-коды.
8. Выберите `Apply Changes`.
9. Вернитесь в игру.

Если чит не работает:

- проверьте, подходит ли он к вашему ядру
- проверьте, подходит ли он к версии ROM
- проверьте регион ROM

## 30. Темы, музыка, заставка и медиа можно добавлять вручную

Это нормальный способ работы с системой.

Куда что класть:

- музыка: `/music`
- картинки заставки: `/screensaver`
- темы ES-DE: `/esde/themes`
- обложки и прочее медиа: `/esde/downloaded_media`
- читы: `/retroarch/cheats`

## 31. Что не стоит менять вручную

Без необходимости не меняйте:

- `/retroarch/cores`
- `/retroarch/info`
- `/esde/resources`
- системные файлы на `rootfs`

Если вы хотите просто добавить контент, обычно достаточно работать только с:

- `/roms`
- `/bios`
- `/music`
- `/screensaver`
- `/esde/themes`
- `/esde/downloaded_media`
- `/retroarch/cheats`

## 32. Если ES-DE не видит новые игры

Проверьте по порядку:

1. ROM лежит в правильной системной папке.
2. Файл не повреждён.
3. Для системы не нужен отсутствующий BIOS.
4. После копирования вы перезапустили ES-DE.
5. Папка системы действительно поддерживается этой сборкой.

## 33. Если игра видна, но не запускается

Самые частые причины:

- отсутствует BIOS
- BIOS назван неправильно
- ROM неподходящего формата
- игра повреждена
- игра положена не в ту системную папку

## 34. Если описание видно, а картинки не видны

Проверьте:

1. `gamelist.xml` лежит в `/roms/<system>/gamelist.xml`
2. медиа лежит в `/esde/downloaded_media/<system>/...`
3. имя медиа совпадает с именем ROM без расширения
4. если ROM лежит в подпапке, структура подпапок повторена в `downloaded_media`
5. после копирования вы перезапустили ES-DE

## 35. Если не работает геймпад

Проверьте:

1. видит ли его сама система
2. завершён ли мастер настройки в ES-DE
3. не повреждён ли файл `/esde/settings/es_input.xml`

Если нужно начать заново:

- удалите `/esde/settings/es_input.xml`
- перезапустите ES-DE

## 36. Если пропала музыка в ES-DE

Проверьте:

1. включён ли пункт `ENABLE BACKGROUND MUSIC`
2. не стоит ли громкость `BACKGROUND MUSIC VOLUME` на нуле
3. лежат ли файлы в `/music`
4. подходят ли форматы файлов

## 37. Если не срабатывает screensaver

Проверьте:

1. есть ли картинки в `/screensaver`
2. корректны ли форматы `.jpg` или `.png`
3. задан ли таймер в `SCREENSAVER SETTINGS`
4. выбран ли тип `SLIDESHOW`
5. включены ли `USE CUSTOM IMAGES`

## 38. Если нужно быстро понять, куда что класть

Короткая памятка:

- игры: `/roms`
- BIOS: `/bios`
- музыка ES-DE: `/music`
- картинки заставки: `/screensaver`
- темы ES-DE: `/esde/themes`
- медиа игр: `/esde/downloaded_media`
- читы RetroArch: `/retroarch/cheats`
- обычные сохранения: `/retroarch/saves`
- save states: `/retroarch/states`
- скриншоты: `/retroarch/screenshots`

## 39. Минимальный сценарий для пользователя

Если не хотите вникать в детали, делайте так:

1. Запишите образ на карту.
2. Включите стик.
3. Дождитесь завершения первого этапа.
4. Включите стик повторно.
5. Дождитесь второго этапа и загрузки ES-DE.
6. Выключите стик.
7. Откройте `USERDATA` на компьютере.
8. Скопируйте игры в `/roms`.
9. Если нужно, скопируйте BIOS в `/bios`.
10. Если хотите, добавьте музыку в `/music`.
11. Если хотите, добавьте картинки заставки в `/screensaver`.
12. Если хотите, добавьте темы в `/esde/themes`.
13. Вставьте карту обратно.
14. Запустите стик.
15. Если новые игры или медиа не появились сразу, перезапустите ES-DE через `REBOOT SYSTEM`.

## 40. Итог

GameStick OS рассчитан на то, что почти вся пользовательская работа идёт через раздел `USERDATA`.

В повседневном использовании вам обычно достаточно знать только несколько папок:

- `/roms`
- `/bios`
- `/music`
- `/screensaver`
- `/esde/themes`
- `/esde/downloaded_media`
- `/retroarch/cheats`

Если соблюдать структуру папок и не трогать системные ядра вручную, система остаётся предсказуемой и удобной в обслуживании.

## 41. Актуальные hotkeys

Ниже приведён актуальный набор hotkeys для текущей версии GameStick OS.

Важно:

- для комбинаций с `HOTKEY` нужно один раз назначить кнопку `HOTKEY / GUIDE` в мастере настройки геймпада `ES-DE`
- если эта кнопка не назначена, будут работать не все сервисные комбинации

### 41.1. ES-DE

- `L3`
  Переключить на предыдущий трек фоновой музыки.

- `R3`
  Переключить на следующий трек фоновой музыки.

- `HOTKEY / GUIDE + левый стик вверх`
  Увеличить громкость фоновой музыки.

- `HOTKEY / GUIDE + левый стик вниз`
  Уменьшить громкость фоновой музыки.

### 41.2. RetroArch

- `HOTKEY / GUIDE + SELECT`
  Открыть меню `RetroArch`.

- `HOTKEY / GUIDE + START`
  Быстро закрыть игру и вернуться в `ES-DE`.

- `HOTKEY / GUIDE + L1`
  Быстро загрузить save state.

- `HOTKEY / GUIDE + R1`
  Быстро сохранить save state.

### 41.3. Fallback

- `L3 + R3`
  Открыть меню `RetroArch`, если кнопка `HOTKEY / GUIDE` не назначена.

## 42. Как пользоваться save state

Save state и обычные сохранения игры — это не одно и то же.

- обычные сохранения хранятся в:
  - `/retroarch/saves`
- save state хранятся в:
  - `/retroarch/states`

Что важно:

- обычный save внутри игры работает так, как это задумал сам разработчик игры
- save state делает мгновенный снимок текущего состояния эмулятора

Как пользоваться:

1. Запустите игру.
2. Когда хотите сохранить мгновенное состояние, нажмите:
   - `HOTKEY / GUIDE + R1`
3. Когда хотите быстро загрузить последнее состояние, нажмите:
   - `HOTKEY / GUIDE + L1`

Если хотите работать через меню:

1. Откройте меню `RetroArch` комбинацией:
   - `HOTKEY / GUIDE + SELECT`
2. Перейдите в `Quick Menu`.
3. Используйте пункты `Save State` и `Load State`.

## 43. Как работают многодисковые игры через M3U

Многодисковые игры у нас поддерживаются.

Смысл такой:

- в папке игры лежат реальные образы дисков
- рядом лежит один файл `.m3u`
- именно файл `.m3u` показывается в `ES-DE` как основная игра

Пример структуры:

```text
/roms/psx/Final Fantasy VII/
  Final Fantasy VII (Disc 1).chd
  Final Fantasy VII (Disc 2).chd
  Final Fantasy VII (Disc 3).chd
  Final Fantasy VII.m3u
```

Содержимое файла `.m3u`:

```text
Final Fantasy VII (Disc 1).chd
Final Fantasy VII (Disc 2).chd
Final Fantasy VII (Disc 3).chd
```

Правила:

- файл `.m3u` должен ссылаться на файлы по относительным путям
- имена должны точно совпадать
- все перечисленные файлы должны реально лежать рядом или по указанному пути

Как запускать:

- в `ES-DE` запускайте именно файл `.m3u`, а не отдельный диск

Как менять диск во время игры:

1. Откройте меню `RetroArch`:
   - `HOTKEY / GUIDE + SELECT`
2. Перейдите в:
   - `Quick Menu -> Disc Control`
3. Выполните:
   - `Eject Disc`
4. Выберите нужный диск
5. Выполните:
   - `Insert Disc`

Где это особенно актуально:

- `PSX`
- другие дисковые системы, где в `ES-DE` разрешены `.m3u`

Практический совет:

- для одиночных игр лучше хранить один `.chd`
- `.m3u` используйте именно там, где у игры два и более диска
